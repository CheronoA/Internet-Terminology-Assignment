🖥️ Web Servers

A web server is hardware or software that stores, processes, and delivers web pages to clients (browsers) over HTTP/HTTPS. It listens on particular ports for incoming requests and responds with files or dynamically generated content.

🌍 Popular Web Servers

🐘 Apache
⚡ Nginx
🪟 IIS (Microsoft)
☕ Tomcat
🚀 Node.js

🔄 How a Web Server Works

  1. Listen - Server waits for HTTP requests on port 80 (HTTP) or 443 (HTTPS)
  2. Receive - Browser sends a request (e.g., GET /index.html)
  3. Process - Server maps URL to file or executes server-side code
  4. Respond - Server sends HTTP response with status code and content
  5. Log - Request is recorded in server logs

📋 Example Request & Response

Client Request:
GET /home.html HTTP/1.1
Host: www.example.com
User-Agent: Chrome/120.0

Server Response:
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html><body><h1>Welcome!</h1></body></html>

🔧 Types of Web Servers

  • Static Web Server - Serves pre-built HTML/CSS/JS files
  • Dynamic Web Server - Generates content on-the-fly (PHP, Python, Node.js)
  • Application Server - Runs business logic and serves APIs

📊 Popular Ports

  • Port 80 - Default for HTTP
  • Port 443 - Default for HTTPS (encrypted)
  • Port 21 - FTP (file transfer)
  • Port 22 - SSH (secure shell)
