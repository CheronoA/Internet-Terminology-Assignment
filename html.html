📝 HTML — The Skeleton of Every Webpage

HTML (Hypertext Markup Language) is the language used to create webpages. Every single website you've ever visited — Google, YouTube, Instagram, this page — is built with HTML. It's not a programming language (it doesn't do logic or math); it's a markup language that describes the structure of a page.

✏️ My simple way to think about it: If a webpage were a house, HTML would be the bricks, walls, and roof. CSS would be the paint and furniture (styling). JavaScript would be the electricity and plumbing (interactivity). HTML is the foundation — without it, nothing else matters.

🏗️ Basic HTML Structure

<!DOCTYPE html>
<html>
  <head>
    <title>My Webpage</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    <p>This is my first webpage.</p>
  </body>
</html>

🔖 Common HTML Tags I Use

<h1> to <h6> <p> <a> <img> <ul>, <ol>, <li> <div> <span> <form> <input> <button> <table> <header> <footer> <nav> <section>

✨ What Each Tag Does

  • <h1> to <h6> — headings (h1 is biggest, h6 is smallest)
  • <p> — paragraph (for normal text)
  • <a> — anchor (creates clickable links)
  • <img> — image (displays pictures)
  • <ul>, <ol>, <li> — bullet lists and numbered lists
  • <div> — a container (groups other elements together)
  • <form>, <input> — for collecting user input (search bars, login forms)

🆚 HTML4 vs HTML5

HTML5 is the current version. It added:

  • Semantic tags — <header>, <footer>, <nav>, <article> (makes code easier to read)
  • <video> and <audio> — no more Flash or plugins needed
  • <canvas> — for drawing graphics with JavaScript
  • Local storage — websites can save data on your computer
💭 What I think: When I first saw HTML, I thought it looked messy with all the angle brackets. But once you understand opening tags <tag> and closing tags </tag>, it starts to make sense. It's actually pretty logical. And you can learn the basics in a few hours — which is why the web grew so fast.
