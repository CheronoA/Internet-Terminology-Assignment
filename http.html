📡 HTTP (Hypertext Transfer Protocol)

HTTP is the application-layer protocol used to transfer data on the World Wide Web. It defines how requests are formatted by clients (browsers) and how servers respond.

🔑 Key HTTP Methods

GET - Retrieve data from the server
POST - Submit data to be processed
PUT - Update existing resource
DELETE - Remove a resource

📊 HTTP Status Codes

200 OK 301 Redirect 404 Not Found 500 Server Error

🔄 HTTP Request-Response Cycle

The request-response cycle is the fundamental mechanism of the web. Every time a user visits a page, the browser and server go through the following steps:

  1. User enters URL - The user types a URL or clicks a link in the browser
  2. DNS resolution - The browser queries DNS to translate domain name to IP address
  3. TCP connection - Browser establishes TCP connection with server (3-way handshake)
  4. HTTP request sent - Browser sends HTTP request with method, path, headers
  5. Server processes request - Web server executes application logic (database queries, authentication)
  6. HTTP response returned - Server sends back status code, headers, and response body
  7. Browser renders page - Browser parses HTML, builds DOM, paints the page
  8. Connection closed/kept alive - TCP connection is closed or kept for future requests

🔒 HTTPS (Secure HTTP)

HTTPS adds encryption (SSL/TLS) on top of HTTP, securing data transmission between browser and server. It protects against eavesdropping and man-in-the-middle attacks.

📋 Example HTTP Request

GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Chrome/120.0
Accept: text/html

📋 Example HTTP Response

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
