🔗 URL — How We Find Things Online

A URL (Uniform Resource Locator) is basically the address of a webpage. Just like your house has a street address so people can find you, every webpage has a URL so your browser can find it. You see URLs every day in your browser's address bar — that's where you type to go to a website.

✏️ My simple way to think about it: A URL is like giving directions to a friend. You tell them the street name, the building number, which floor to go to, and maybe a landmark. The URL does the same thing — it tells your browser exactly where to go on the internet.

📋 URL Structure — Breaking It Down

https://www.example.com:443/products/item123?id=42&color=red#reviews
1. Scheme/Protocol (https://)
This tells the browser how to connect. "https://" means secure web traffic. "http://" is regular (less secure). "ftp://" is for file transfers. It's like saying "use the front door" vs "use the back door."
2. Subdomain (www)
An optional prefix that tells the browser which part of the website you want. "www" is the most common, but you also see "blog." (blog.example.com), "shop." (shop.example.com), or "mail." (mail.example.com).
3. Domain Name (example.com)
The main name of the website. This is what you usually remember (google.com, youtube.com, wikipedia.org). It has two parts: the name ("google") and the extension (".com", ".org", ".ke").
4. Port Number (:443)
An optional technical detail. Most of the time you don't see it because browsers use defaults (80 for HTTP, 443 for HTTPS). It's like saying "deliver to apartment number 443." You usually don't need to type it.
5. Path (/products/item123)
This tells the server which specific page or file you want. It's like telling someone which room to go to after they enter the building. The path often matches the folder structure on the server.
6. Query String (?id=42&color=red)
Extra information sent to the server, usually for search results or filters. It starts with a "?" and separates parameters with "&". For example, "?q=cats" means search for cats. "?page=2&sort=newest" means go to page 2, sorted by newest.
7. Fragment (#reviews)
Points to a specific section inside a webpage. For example, if a page has a "Reviews" section, "#reviews" will scroll you straight to it. The fragment never gets sent to the server — it's only used by your browser.
