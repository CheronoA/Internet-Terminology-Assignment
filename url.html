🔗 URL (Uniform Resource Locator)

A URL is a reference (address) to a specific resource on the web. It specifies the protocol, host, path, and optionally query parameters or fragments, allowing browsers to locate and retrieve any resource uniquely.

📋 URL Structure Breakdown

https://www.example.com:443/products/item123?id=42&color=red#reviews
1. Scheme/Protocol (https://)
How to access the resource. Common schemes: http, https, ftp, file, mailto.
2. Subdomain (www)
Optional prefix identifying a service or section within the domain (e.g., blog., shop., mail., api.)
3. Domain Name (example.com)
Human-readable host address (the website name). The domain name includes the second-level domain (example) and top-level domain (.com).
4. Port Number (:443)
Network endpoint (optional; default: 80 for HTTP, 443 for HTTPS). Other ports: 21 (FTP), 22 (SSH), 3306 (MySQL).
5. Path (/products/item123)
Location of the resource on the server (similar to file path on a computer). Can be actual file path or virtual path processed by server.
6. Query String (?id=42&color=red)
Key-value parameters for dynamic pages (e.g., search results, filters, pagination). Starts with ? and separates parameters with &.
7. Fragment (#reviews)
Points to a specific section within the page (anchor link). Does not send to server — browser only. Used for in-page navigation.

📊 URL Encoding

Some characters must be encoded in URLs: space becomes %20, # becomes %23, & becomes %26, / becomes %2F.
